시네마틱 컨셉보드 전용 Gemini Gems 제작 실습

4일차 12교시
매번 복잡한 카메라 구도·조명· 지시어를 길게 쓰는 번거로움을 해결하기 위해, 장면 키워드만 넣어도 이미지 AI에 최적화된 명세를 정교하게 뱉어내는 / Custom GPTs의 를 생성합니다.
Gems 만들기 — 천천히 따라하기
1
gemini.google.com 접속 → 왼쪽 메뉴 'Gem 관리자''새 Gem 만들기' 버튼을 누릅니다.
2
이름 입력 후 '지침(Instructions)' 칸에 붙여넣기아래에서 테마를 고르고 생성된 시스템 명세를 [복사] → 그대로 붙여넣습니다.
3
저장 → 테스트"사무실 야근 장면"처럼 짧은 키워드만 입력해 보세요. ① 3단 연출 명세(텍스트 컨셉보드)와 ② 그 프롬프트로 즉시 생성된 이미지가 한 답변에 함께 나오면 성공입니다. (지침 4번 'Image Render' 규칙 덕분에 Gem이 이미지까지 직접 그려줍니다)
원하는 기획 시나리오 테마 선택
A·B·C와 D의 차이 A·B·C는 이미지 컨셉보드(장면 1개 → 연출 명세 + 이미지 1장)이고, D는 영상 스토리보드 풀패키지입니다. D에 주제와 영상 길이만 입력하면 장면별로 연출 방향 · 카메라 무브먼트 · 화면 묘사 · 대사/내레이션 · 자막 · BGM/효과음 제안 · 이미지 프롬프트 + 장면 이미지까지 한 세트로 나옵니다. 장면 이미지는 실제 영화 현장 스토리보드처럼 연필 스케치(흑백 그래파이트) 스타일로 통일되어 그려집니다 — 사진풍이 아니라 스케치북에 그린 콘티 느낌이라, 회의·수정 단계에서 부담 없이 쓰기 좋습니다. 출력 형식은 장면마다 [스케치 이미지 → 바로 아래 콘티 상세표]가 한 쌍(시트)으로 반복되는 구조입니다. Gemini 채팅 화면 특성상 이미지를 표 안에 넣는 완전한 좌우 2단 배치는 불가능해서, 인쇄된 콘티북처럼 '스케치 페이지 + 콘티 페이지'가 짝지어 나오는 방식이 가장 가까운 형태입니다. 인쇄용 좌우 배치가 필요하면 결과를 복사해 구글 문서 2단 표에 붙여 정리하면 됩니다. ※ BGM·효과음은 '텍스트 제안'입니다(실제 음원 생성 아님). 장면 이미지가 많으면 생성 한도에 걸릴 수 있어, 지침이 핵심 장면부터 순서대로 그리도록 설계되어 있습니다. 완성된 스토리보드는 3교시 구글 Flow에 컷별로 넣어 실제 영상으로 만듭니다.
왜 Gems로 만들까요? 한 번 규칙을 심어두면 팀원 누구나 "장면 키워드"만 입력해서 연출 명세 + 완성 이미지를 한 번에 받을 수 있습니다. 우리 회사 전용 '숙련된 디자이너 조수'를 만드는 셈입니다. ※ 지침 4번(Image Render) 규칙이 이미지 동시 생성을 담당합니다. Gemini의 이미지 생성 한도(무료 계정 일일 제한)에 걸리면 그날은 ③번 프롬프트만 출력되며, 이 경우 프롬프트를 복사해 다른 이미지 도구에 붙여넣으면 됩니다. 실존 인물 얼굴 등은 정책상 생성이 제한될 수 있습니다.

GEMS DESIGN RULES

컨셉보드 Gems 설계 시 핵심 주의점

  • 화면 일관성: 다른 장면에서도 동일한 인물(또는 의자)이 연속 등장할 수 있게 'consistent character description' 형태의 연속 규칙 지정.
  • 가짜 글자 원천 예방: 텍스트가 무의미하게 기입되지 않도록 'no gibberish AI overlay text' 규칙을 강하게 기입.
  • 화면 비율 고정: 가로형 시네마틱 컷은 16:9, 소셜 피드는 1:1로 사전 고정 출력 유도.
Gems/GPTs 환경에서 '지침(Instructions)' 탭에 이 명세를 붙여 넣으면 실습생 전용 지능형 에이전트가 생성됩니다.
